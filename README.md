# Гайд для домохозяек
Для начала работы требуется открыть ваш терминал, перейти в корневую папку проекта и ввести команду ```cmake . && make```.
Теперь вы можете запустить автомат командой

## Операции с NFA
```bash
./nfa -f <путь файла с кодом автомата> -s "<входная строка>"
```
вы так же можете использовать флаг `-ds` вместо `-s`, тогда строка будет читаться посимвольно, а не по числам.

Чтобы получить детерминизированную версию введенного NFA используйте команду
```bash
./nfa -f <путь входного файла с кодом NFA> -d <путь выходного файла с кодом DFA>
```

Пример:

```bash
./nfa -f sample_nfa/mult5.txt -s "1 0 1"
```
или

```bash
./nfa -f sample_nfa/mult5.txt -ds "101"
```

Запускает автомат на строке "1 0 1", выводит ```true```.
```bash
./nfa -f sample_nfa/big_nfa.txt -d big_dfa.txt
```
Детерминизирует автомат и сохраняет результат в ```big_dfa.txt```.

## Примеры NFA
В папке ```sample_nfa``` лежит 4 файла с автоматами.
- ```simple_nfa.txt``` пример NFA.
- ```big_nfa.txt``` пример большого NFA, принимает строки, в которых количество или 0, или 1, или 2 кратно 4.  
- ```odd01.txt``` принимает строки, в которых количество и 0, и 1 нечётно.
- ```mult5.txt``` принимает строки, числовое представление которых кратно 5.

## Операции с DFA

#### Минимизация
```bash
./nfa -f <путь входного файла с кодом DFA> -m <путь выходного файла с кодом нового DFA>
```

Вернет вам минимальный автомат, принимающий тот же язык, что и DFA во входном файле. 
Важно, чтобы данный автомат был именно DFA, иначе корректность работы не гарантирована.

#### Проверка языков на эквивалентность 
```bash
./nfa -f <путь первого файла с кодом DFA> -e <путь второго файла с кодом DFA>
```

Проверяет, что языки, задаваемые данными автоматами эквивалентны.
Так же есть обертка над этой функцией:

```bash
./nfa -f <путь первого файла с кодом DFA> -a
```

сообщит, принимает ли этот автомат любую строку заданную на алфавите входного автомата.

## Регулярные выражения

Вы можете преобразовать регулярное выражение над алфавитом из цифр от 0 до 9, в NFA, используя команду

```bash
./nfa -f <файл с regEx-ом> -re <выходной файл с кодом NFA>
```

Данный NFA будет принимать язык, заданный вашим регулярным выражением. 
Вы так же можете детерминизировать его, и затем минимизировать, тогда он будет работать быстрее, но может занимать $O(2^n)$ памяти.
